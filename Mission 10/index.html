Mission 10 : Conception Base de Données Cinéma

1. Optimisation du modèle de données

L'analyse du schéma initial a révélé une anomalie : la table film comportait le nom du pays en toutes lettres ("France", "Etats-Unis"), ce qui créait une redondance avec la table pays existante.
Action corrective : J'ai remplacé la colonne texte par une clé étrangère (IDENT_PAYS) de type entier, pointant directement vers la table de référence.

2. Implémentation technique

La base mission_cinema_ugc a été structurée autour de 5 entités :

  • Entités principales : Film, Acteur, Réalisateur, Pays.
  • Relation : La table casting fait le lien entre les films et les acteurs (clé primaire composite).

Les contraintes d'intégrité (Foreign Keys) ont été activées pour garantir la cohérence des données lors des insertions.

3. Requêtes d'extraction SQL

Recherche multicritères (Durée et Date)

Critères : Films > 90 min sortis entre 2008 et 2012.

SELECT TITRE FROM film WHERE DUREE > 90 AND DATE_SORTIE BETWEEN '2008-01-01' AND '2012-12-31';

Résultat obtenu : AVATAR

Filtrage sur données numériques

Objectif : Lister les acteurs ayant une filmographie comprise entre 40 et 90 titres.

SELECT NOM, PRENOM FROM acteur WHERE NB_FILM BETWEEN 40 AND 90;

4. Mise à jour de la base

Pour l'ajout du film "Bienvenue chez les Chti's", une double insertion a été nécessaire : création du réalisateur Dany Boon, puis insertion du film en référençant ce nouveau réalisateur et le pays France (ID 1).
